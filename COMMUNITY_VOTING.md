# 🗳️ Mck Skill Hub — 社区技能提案与投票规则

> **版本**: v1.0 | **生效日期**: 2026 年 3 月 | **维护者**: Kaku

---

## 📋 概述

Mck Skill Hub 采用**社区驱动**的技能上架机制。任何成员均可在微信群中提案（Propose）新技能，经腾讯文档投票达到门槛后，技能正式收录进 Gallery。

**核心原则**：低门槛提案 → 透明投票 → 高质量收录

---

## 🔄 完整流程

```
提案 → 审核 → 投票 → 收录 → 发布
 (微信群)  (管理员)  (腾讯文档)  (PR)    (GitHub)
```

### Step 1：提案（Propose）

**在哪里提**：微信群

**提案格式**（直接发群消息）：

```
🆕 技能提案

📛 技能名称：[名称]
📁 象限：[空心胜利 / 有意义的影响 / 漂流 / 安静的满足]
📝 一句话描述：[这个技能解决什么问题]
🔗 链接/出处：[GitHub 链接 / 自己写的 / 发现的]
👤 提案人：[你的名字]

💡 为什么值得收录：
[2-3 句话说明这个技能的价值]
```

**提案来源**（两种都欢迎）：
- 🔍 **发现型** — 你发现了别人做的好用的 AI skill
- 🛠️ **原创型** — 你自己做了一个 AI skill

### Step 2：管理员初审（24 小时内）

管理员收到提案后进行**快速初审**：

| 检查项 | 标准 |
|--------|------|
| 象限归属 | 是否明确属于某个象限 |
| 不重复 | 与已有技能不重叠 |
| 可执行 | 有明确的 SKILL.md 或可落地方案 |
| 合规性 | 无侵权、无有害内容 |

- ✅ **通过** → 进入投票环节
- ❌ **不通过** → 群里回复原因，提案人可修改后重新提交
- ❓ **需补充** → @提案人 补充信息

### Step 3：腾讯文档投票

**投票工具**：腾讯文档 → 投票功能

管理员初审通过后：

1. 管理员在**腾讯文档**中创建投票
   - 标题：`[技能名称] — 是否收录进 Mck Skill Hub？`
   - 选项：`✅ 赞成收录` / `❌ 暂不收录`
   - 设置：**实名投票**、**每人限投一次**
2. 将投票链接发到微信群，@所有人
3. 投票周期：**72 小时**（3 天）

**投票门槛**：

| 条件 | 要求 |
|------|------|
| **赞成票数** | ≥ **10 票** |
| **赞成比例** | ≥ **60%** |
| **投票周期** | 72 小时 |

> 两个条件**同时满足**才算通过。

### Step 4：收录（Merge）

投票通过后：

1. **有 GitHub 仓库的技能**：
   - 提案人（或管理员）向 mck-skill-hub 提交 PR
   - 在 `skills/[象限]/[技能名称]/` 下创建 `SKILL.md` + `README.md`
   - 遵循 [skill-template](template/skill-template/) 格式
   - 管理员审核后合并

2. **暂无仓库的技能**：
   - 管理员代为创建目录和基础 `README.md`
   - 标注 `🧪 Beta` 或 `🔜 Planned`
   - 鼓励社区成员认领完善

3. 更新 `skills.csv` 技能索引

### Step 5：发布（Announce）

- 管理员在微信群发布收录公告
- 更新 GitHub README 技能列表
- @提案人 表示感谢 🎉

## 📊 投票规则细则

### 谁可以投票？

微信群内的**所有成员**均可投票（一人一票）。

### 谁可以提案？

微信群内的**所有成员**均可提案，没有限制。

### 投票频率

- 每周最多同时进行 **3 个投票**（避免投票疲劳）
- 同一技能被否决后，**30 天内不得重新提案**（可修改后重新提）

### 特殊情况

| 场景 | 处理方式 |
|------|----------|
| 72h 后赞成票 < 10 | 投票失效，可 30 天后重新提案 |
| 赞成票 ≥ 10 但比例 < 60% | 投票失效，群里讨论改进后可重新提 |
| 提案人自己的原创技能 | 提案人的赞成票**计入**总数（鼓励原创） |
| 投票期间发现重大问题 | 管理员可提前终止投票并说明原因 |

## 🏷️ 技能质量等级

收录后的技能根据完成度标注等级：

| 标识 | 等级 | 标准 |
|:-----|:-----|:-----|
| `✅ Production` | 生产就绪 | 有完整 SKILL.md + README，经过实际项目验证 |
| `🧪 Beta` | 测试阶段 | 核心逻辑可用，还在收集反馈 |
| `🔜 Planned` | 计划中 | 已通过投票但技能还未完全实现 |

## 📎 附录

### 腾讯文档投票设置模板

创建投票时的标准配置：

- **投票类型**：单选
- **选项**：`✅ 赞成收录` / `❌ 暂不收录`
- **设置**：
  - ☑️ 实名投票
  - ☑️ 每人限投一次
  - ☑️ 截止时间：创建后 72 小时
  - ☑️ 投票后可查看结果

### 微信群公告模板

**发起投票时**：

```
📢 新技能投票开启！

技能：[技能名称]
象限：[象限名]
提案人：[名字]
描述：[一句话]

🗳️ 投票链接：[腾讯文档链接]
⏰ 截止：[日期时间]

规则：≥10 票赞成 + ≥60% 赞成率 = 收录
```

**投票通过时**：

```
🎉 投票通过！

技能「[名称]」已正式收录进 Mck Skill Hub！
赞成：[X] 票 (XX%)
感谢 @提案人 的贡献！

🔗 GitHub：[链接]
```
